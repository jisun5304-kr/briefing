# 과거 브리핑 해설 생성 지침 — 정직형(사후 보정)

이건 **과거 특정일의 사후 재구성** 브리핑이다. 실시간 뉴스가 없고(`news`는 비어 있음),
지수·환율 숫자만 실제값으로 주어진다. 너는 그 **가격 흐름에만 근거해서** 해설을 쓴다.

## 작업
1. `build/input.json`을 읽는다. (특정 과거일의 실제 지수·환율 값. `backfill: true`, `news: []`)
2. 아래 스키마대로 `build/prose.json` **하나만** 작성한다.

## 정직성 규칙 (매우 중요)
- **검증 불가능한 사실을 단정하지 마라.** 그날의 구체적 뉴스·사건·정책·인물·기업별 호재/악재를 **지어내지 마라.** 너는 그날 실제로 무슨 뉴스가 있었는지 모른다.
- **오직 숫자가 말해주는 것만** 서술한다: 어느 지수가 얼마나 올랐/내렸는지, 환율 방향, 미국↔한국 흐름의 대비, 변동성의 크기 등.
- 추론은 추론으로 표현한다: "~로 보인다", "~흐름", "기술적으로". 단정형 사실("A사가 B를 발표해 급등")은 금지.
- `④ 섹터`는 **개별 종목명을 사실처럼 단정하지 말고**, 지수 흐름에서 합리적으로 추정되는 **넓은 부문**(예: 대형 기술주, 반도체 섹터, 수출주, 금융주)으로 쓴다. 방향성은 해당일 지수 방향과 일관되게.
- 과거형으로 서술한다(이미 지나간 날의 정리).
- 한국어, 간결한 톤.

## 출력 스키마 (`build/prose.json`)
```json
{
  "sentiment": "up | down | flat",
  "headline": "📊 그날 지수 흐름을 요약한 한 줄 (가격 사실 기반)",
  "summary": "3~4문장. input.json의 실제 등락률을 인용해 그날의 시장 움직임을 정리. 뉴스 원인 단정 금지, 흐름 위주.",
  "watch_points": "📌 그날 특징: (예: 변동성 확대 / 미국·한국 디커플링 / 환율 방향 등 — 숫자 기반)",
  "index_notes": {
    "S&P 500": "짧은 가격 코멘트", "나스닥": "...", "다우존스": "...",
    "코스피": "...", "코스닥": "...", "USD/KRW": "..."
  },
  "drivers": [
    { "emoji": "📈", "title": "가격 흐름 기반 관찰", "desc": "숫자로 확인되는 사실만" }
    // 4~5개. 뉴스 고유명사 없이, 지수·환율 움직임 위주.
  ],
  "sectors": [
    { "name": "대형 기술주", "reason": "나스닥 흐름상 추정", "direction": "up|down|flat", "direction_text": "▲ 강세 추정" }
    // 3~5개. 넓은 부문 + '추정' 명시.
  ],
  "checklist": [
    { "title": "그날 기준 점검 포인트", "desc": "숫자 기반 한 줄" }
    // 3~4개.
  ]
}
```

## 출력 형식
- 순수 JSON만. 코드펜스·설명문 금지. `{`로 시작해 `}`로 끝낼 것.
- `index_notes` 키는 input.json의 지수 이름과 정확히 동일하게.
