# 시황 브리핑 해설 생성 지침 (Claude Code 용)

너는 한국 주식시장 시황 브리핑의 **해설·전망 부분만** 작성한다.
지수·환율 숫자는 이미 코드가 실제 데이터로 채웠으니, 너는 그 데이터를 **해석**만 한다.

## 작업 순서
1. `build/input.json` 파일을 읽는다. (오늘의 실제 지수·환율 + 수집된 뉴스 제목들이 들어 있다)
2. 아래 스키마에 맞춰 한국어 해설을 작성한다.
3. 결과를 **`build/prose.json`** 파일 하나로만 저장한다. 다른 파일은 만들거나 수정하지 않는다.

## 절대 규칙
- **숫자를 만들어내지 마라.** 지수 값·등락률·환율은 `input.json`에 있는 것이 전부이며, 너는 그것을 인용·해석만 한다. prose.json에는 지수 숫자 필드 자체가 없다.
- 출력은 **순수 JSON만.** 코드펜스(```), 설명 문장, 주석을 붙이지 마라. `build/prose.json` 내용은 `{` 로 시작해 `}` 로 끝나야 한다.
- 뉴스 제목(`input.json`의 `news`)을 근거로 `drivers`·`sectors`를 작성하되, 확인되지 않은 사실을 단정하지 마라. 추측은 "전망"·"가능성" 등으로 표현한다.
- 세션(`session_ko`: 오전/오후/밤)과 기준(`session_basis`)에 맞는 톤으로 쓴다. 오전은 밤사이 미국장 중심, 오후는 국내 마감 중심, 밤은 미국 개장 중심.
- 모든 텍스트는 한국어. 간결한 증권사 데일리 톤(기존 브리핑 스타일).

## 출력 스키마 (`build/prose.json`)
```json
{
  "sentiment": "up | down | flat",   // 오늘 시장 전반 분위기 (① 카드 색상에 사용)
  "headline": "한 줄 핵심 제목 (이모지 1개로 시작, 예: 📈 ...)",
  "summary": "3~5문장 요약. 지수 흐름과 오늘 시장의 핵심 맥락을 서술. input.json의 실제 수치를 자연스럽게 인용.",
  "watch_points": "✅ 오늘 관전 포인트: 항목1, 항목2, 항목3",
  "index_notes": {
    "S&P 500": "한 줄 코멘트(예: 8주 연속 상승)",
    "나스닥": "...",
    "다우존스": "...",
    "코스피": "...",
    "코스닥": "...",
    "USD/KRW": "..."
  },
  "drivers": [
    { "emoji": "🇺🇸", "title": "변수 제목", "desc": "한 줄 설명" }
    // 5~6개. 뉴스에 근거.
  ],
  "sectors": [
    { "name": "삼성전자", "reason": "주목 이유", "direction": "up|down|flat", "direction_text": "▲ 강세 전망" }
    // 4~6개 종목/섹터.
  ],
  "checklist": [
    { "title": "체크 항목", "desc": "왜 중요한지 한 줄" }
    // 3~5개.
  ]
}
```

## index_notes 주의
- 키는 `input.json`의 지수 이름과 **정확히 동일**해야 한다(S&P 500, 나스닥, 다우존스, 코스피, 코스닥, USD/KRW).
- 값은 짧은 코멘트만. 숫자를 새로 쓰지 말고 흐름·맥락만.
- `direction`/`direction_text`의 화살표 방향은 의미상 맞게: 강세 ▲(up), 약세 ▼(down), 중립 →(flat).
